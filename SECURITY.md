# Security Policy

## Supported versions

At this stage, Urlopomat is in active early development.

Security fixes are currently applied only to the latest development version available in the default branch.

## Reporting a vulnerability

If you discover a security vulnerability, please do **not** open a public issue.

Instead, report it privately by email:

**krystian@jablonowski.eu**

Please include:
- a clear description of the issue,
- steps to reproduce,
- potential impact,
- suggested mitigation, if available.

## Response process

When a valid security report is received, the maintainer will aim to:

1. confirm receipt of the report,
2. assess the severity and scope,
3. prepare a fix,
4. publish the fix in a responsible manner.

## Disclosure policy

Please allow reasonable time for the issue to be investigated and fixed before making it public.

Public disclosure before a fix is available may put users of the project at risk.

## Scope

This policy applies to:
- the source code of Urlopomat,
- API logic,
- input validation,
- dependency-related security concerns,
- configuration issues that may affect deployed environments.

## Out of scope

Unless explicitly stated otherwise, this policy does not cover:
- vulnerabilities in third-party services not maintained by this project,
- local development environment misconfiguration outside the repository,
- unsupported forks or heavily modified downstream versions.

## Dependencies

Security-related dependency updates may be handled through standard dependency management and repository security tools.
